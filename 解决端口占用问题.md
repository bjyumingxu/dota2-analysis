# 🔧 解决端口占用问题

## 🚨 问题现象
- 即使没有启动后端，`http://localhost:8000/health` 依然能访问
- 说明8000端口被其他服务占用

## 🔍 检查方法

### 方法1：查看端口占用详情

在PowerShell中执行：
```powershell
Get-NetTCPConnection -LocalPort 8000 | Select-Object LocalAddress, LocalPort, State, OwningProcess
```

这会显示占用8000端口的进程ID。

### 方法2：查看进程详情

如果找到了进程ID（例如：12345），执行：
```powershell
Get-Process -Id 12345 | Select-Object Id, ProcessName, Path
```

### 方法3：使用不同端口

如果无法结束占用8000端口的进程，我们可以改用其他端口（例如：8001）

## ✅ 解决方案

### 方案1：改用8001端口（推荐）

修改后端启动命令：
```powershell
uvicorn src.main:app --reload --host 0.0.0.0 --port 8001
```

然后修改前端代理配置：
- 打开 `frontend/vite.config.ts`
- 将 `target: 'http://localhost:8000'` 改为 `target: 'http://localhost:8001'`

### 方案2：找到并结束占用8000端口的进程

1. 找到进程ID：
```powershell
Get-NetTCPConnection -LocalPort 8000 | Select-Object OwningProcess
```

2. 结束进程：
```powershell
Stop-Process -Id <进程ID> -Force
```

3. 如果提示权限不足，以管理员身份运行PowerShell
